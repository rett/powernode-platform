## 💡 Cost Optimization Strategies

### 1. **Model Cascading**
Start with cheaper model, escalate to expensive only if needed:
```
User Query → GPT-4o-mini (try first)
  ↓ (if insufficient)
User Query → Claude Sonnet 4.5 (escalate)
  ↓ (if still insufficient)
User Query → Claude Opus 4.1 (final escalation)
```

### 2. **Parallel Execution**
Use Anthropic's recommended pattern:
```
Planner: Claude Sonnet 4.5 (multi-step planning)
Workers: Pool of Claude Haiku 4.5 (parallel execution)
```
**Cost**: $1.05 + (N × $0.35) vs (N × $1.05)
**Savings**: ~67% on worker tasks

### 3. **Batch Processing**
- OpenAI Batch API: 50% discount on all models
- Anthropic Batch API: 50% discount on all models
- Use for non-urgent tasks with 24-hour SLA

### 4. **Prompt Caching** (Anthropic)
- Cache writes: 1.25x base price
- Cache hits: 0.1x base price (90% savings)
- 5-minute TTL
- Ideal for repeated contexts

### 5. **Right-Sizing Context**
- Don't send full context if not needed
- Use summarization for historical data
- Trim unnecessary context

### 6. **Self-Hosting** (Ollama)
- Zero API costs
- One-time hardware investment
- Good for privacy-sensitive or high-volume
