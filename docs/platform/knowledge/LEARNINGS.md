### Every _id foreign key column MUST have a corresponding belongs_to association — programmatic writes are not exempt [VERIFIED]

CompoundLearning had verified_by_id and disproven_by_id columns written by verify!/disprove! methods, but no belongs_to associations. This meant the foreign keys could not be hydrated via ActiveRecord (learning.verified_by returned nil even with a valid ID). Rule: every _id column that stores a foreign key MUST have a corresponding belongs_to association, even if the column is only written programmatically. Use reflect_on_association in rails console to audit.
